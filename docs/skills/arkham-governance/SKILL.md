# Skill: Arkham Governance

Guidance for agents operating within the Arkham Control Plane to maintain architectural consistency and execute durable operations.

## 1. Architectural Alignment
Every major recommendation or proposal generated by an agent should be checked against the **Arkham Core Principles**:
- **BIM-First:** Structured data over flat files.
- **Durable Orchestration:** State must be persisted at every major step.
- **Read-First, Execute-Later:** Direct mutations must be gated by explicit `WorkflowApprovalState`.

## 2. Managing Workflows
When initiating or resuming work in the `arkham` repo:
1.  **Check Active Workflows:** Query `workflow_runs` to see if a relevant task is already `running` or `paused`.
2.  **Resume vs. Restart:** If a workflow is `paused` with `approval_state=pending`, do NOT restart. Instead, present the artifact to the operator for approval.
3.  **Checkpointing:** Ensure all intermediate state is stored in the `checkpoint` JSONB field of the `WorkflowRunRecord`.

## 3. The "Arkham" Naming Rule
- **Directive:** Ensure all new resources, services, and labels use the `arkham-` prefix.
- **Audit Logic:** Use the `CodebaseAuditWorker` to scan for `robco` legacy refs in Terraform, Cloud Build, and environment variables.
- **Classification:** Tier all findings into `safe`, `controlled`, or `breaking` before presenting a remediation plan.

## 4. Interaction Protocol
- **Gate Enforcement:** For high-stakes mutations (DB, Network, Deploys), use the `transactional_session` to ensure atomicity.
- **Evidence Capture:** Always link `CodebaseAuditRecord` findings to specific line numbers and snippets in the `evidence` field.
